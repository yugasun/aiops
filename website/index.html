Specialized agents
9 agents. Each with identity, skills, and artifact contracts.
Agents are nouns (who does it). Skills are verbs (what to do). Each agent wraps skills with identity, constraints, and defined outputs. They communicate through .scratch/<feature>/ artifact files.
architect
alignment
Design decisions and tech specs. Explores the problem space, scans existing architecture for deepening opportunities, models the domain. Does not write implementation code.
/grilling /grill-with-docs /domain-modeling /architect-design /improve-codebase-architecture
→ NOTES.md · tech-spec.md
design-reviewer
design gate
Independent review of architect's design. Checks design soundness, depth, seam placement, and alignment with domain model. Strong gate — blocks planner until APPROVE.
/review
→ DESIGN_REVIEW.md
planner
planning
Breaks design into vertical slices. Synthesizes PRDs, creates implementation plans with clear acceptance criteria. Manages dependencies between issues.
/to-prd /to-issues /handoff /aiops-setup
→ PRD.md · plan.md · issues/
prototyper
prototype
Rapid validation with throwaway code. Answers "can this work?" in minimal time. Not concerned with code quality — only feasibility.
/prototype /lean
→ VERDICT.md · prototype/
builder
delivery
TDD implementation with hard quality gates. Follows the lean→tdd→prune→review→commit sequence. One issue at a time, no premature abstraction.
/aiops-implement /tdd /lean
→ source code + tests
ui-designer
design
Visual reference without business logic. Produces browser-openable HTML mockups with design decisions documented. No JS event handlers, no build step.
/ui-mockup
→ mockups/ · design-notes.md
code-reviewer
delivery gate
Independent review against design intent. Checks correctness, security, and alignment with NOTES.md. Distinguishes blocking from non-blocking findings.
/review
→ REVIEW.md
quality-auditor
delivery gate
Over-engineering detection. Finds unnecessary abstractions, premature optimization, and scope creep. Uses the lean ladder as the judgment standard. Runs only after code-reviewer approves.
/prune
→ prune findings
gitops
delivery
Version control operator. Commits with Conventional Commits format, traces back to .scratch artifacts. Does not modify code — only git operations.
/gitops
→ commit + push

Multi-IDE portability
One source. Every IDE.
The adapter seam compiles SKILL.md to each IDE's native format at install time. Zero drift between platforms. Adding a new IDE = writing one adapter file.
