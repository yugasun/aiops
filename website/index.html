安装
一行命令
推荐
$ npx -y github:yugasun/aiops
Skills CLI 全局
$ npx skills@latest add yugasun/aiops -g -y --skill '*'
可选 · 代码图谱（graphify）
$ uv tool install graphifyy

graphify 是 Python 开源工具，为 AI 提供结构化代码理解（AST 解析 + 社区检测）。 需要先安装 uv（Python 包管理器）。 不装也能用其他 20 个 skill。

可选：团队用 GitHub / GitLab

默认任务记在 .scratch/。只有需要远程 issue 时，在项目根加 aiops.yaml：

version: 1
issue_tracker:
  kind: github
  prs_as_triage: false

没有 yaml = 本地 markdown，首次 /aiops 不问卷。

快速开始
三步就能开工

零配置默认：本地 markdown 记任务。点下面每一步看说明。

1
安装到 AI IDE
终端一行命令，支持 Cursor、Claude Code、Codex、Copilot、Windsurf。装完重启 IDE。
2
打开项目，输入 /aiops
中文描述目标即可，例如「修登录 500」「给后台加导出」。不用写 Task type。
3
跟着「第 N 步」走
进度在 .scratch/<功能名>/flow.state.yaml，下次 /aiops 继续 接着做。
第 1/3 步 · 安装 (Install)

在终端运行安装命令，重启 IDE 后即可在聊天框使用 /aiops。

无 aiops 时 AI 自行添加的内容（用户没要求）：
- interface HealthStatus { status, version, timestamp, uptime, memory, environment } - router.get("/health/ready", ...) // 用户没要求 - router.get("/health/live", ...) // 用户没要求 // YAGNI ladder + prune 阻止了这些过度工程
有 aiops 时的实现（只返回用户要求的字段）：
+ const { version } = JSON.parse(readFileSync(join(__dirname, "../../package.json"), "utf8")); + router.get("/health", (_req, res) => { res.json({ status: "ok", version }); }); // 14 行，零过度工程，2 个测试全部通过

无 aiops — 直觉修复

diff 行数+1/-1 行
测试结果❌ FAILS
返回码200（静默忽略）
诊断过程无（直觉修复）

有 aiops — TDD 驱动

diff 行数+4/-1 行
测试结果✅ PASSES
返回码400（明确拒绝）
诊断过程假设 → 验证 → 修复
无 aiops 的修复 — 静默 bug（表面修了，测试仍失败）：
- if (req.body.title !== undefined) task.title = req.body.title; + if (req.body.title !== undefined && req.body.title !== "") task.title = req.body.title; // 静默忽略空标题，返回 200 — 测试期望 400，失败
有 aiops 的修复 — TDD 驱动正确实现：
+ if (req.body.title !== undefined) { + if (!req.body.title.trim()) return res.status(400).json({ error: "title cannot be empty" }); + task.title = req.body.title; + } // 明确返回 400，测试通过
